# 법인카드 전표 계정과목 AI 추천 시스템

임베딩 모델 파인튜닝과 Qdrant 벡터 DB를 활용한 법인카드 전표 계정과목 자동 추천 시스템입니다.

## 개요

법인카드 전표 입력 시 과거 전표 데이터를 기반으로 적절한 계정과목을 자동으로 추천합니다.  
단순 키워드 매칭이 아닌 **의미 기반 유사도 검색**을 사용하므로, 학습 데이터에 없는 새로운 가맹점도 유사한 업종으로 추천이 가능합니다.

## 동작 방식

```
전표 텍스트 입력
       ↓
임베딩 모델 (파인튜닝된)
       ↓
벡터로 변환 → Qdrant 저장
       ↓
새 전표 입력 시 유사 전표 검색
       ↓
계정과목 추천
```

## 기술 스택

- **Python 3.10+**
- **sentence-transformers** - 임베딩 모델 및 파인튜닝
- **Qdrant** - 벡터 DB (인메모리 모드)
- **Google Colab** - 실행 환경

## 추천 계정과목 종류

| 계정과목 | 예시 가맹점 |
|---------|-----------|
| 복리후생비 | 스타벅스, 투썸플레이스, 맥도날드 |
| 접대비 | 한우마을, 오마카세, 호텔라운지 |
| 차량유지비 | 현대오일뱅크, SK주유소, 주차장 |
| 도서인쇄비 | YES24, 교보문고, 알라딘 |
| 소모품비 | 오피스디포, 다이소, 쿠팡 |
| 통신비 | KT, SKT, LG유플러스 |

## 실행 방법

1. Google Colab에서 `전표_계정과목_추천_실습.ipynb` 열기
2. 셀 순서대로 실행 (Shift+Enter)
3. 마지막 셀에서 전표 내용 입력 후 추천 결과 확인

```python
계정과목_추천("이디야커피 잠실점 / 팀회식 / 41000원")
# ✅ 추천 계정과목: [복리후생비]

계정과목_추천("에쓰오일 판교점 / 주유 / 68000원")
# ✅ 추천 계정과목: [차량유지비]
```

## 핵심 개념

### 벡터 DB란?
텍스트를 고차원 좌표(벡터)로 변환하여 저장하고, 의미상 가까운 데이터를 빠르게 검색하는 DB입니다.

### 파인튜닝이란?
기본 임베딩 모델이 모르는 업무 도메인 맥락(전표 계정과목)을 추가 학습시켜 검색 정확도를 높이는 작업입니다.

### 데이터가 쌓일수록 정확도 향상
Qdrant에 저장된 전표 데이터가 많아질수록 유사 전표를 더 정확하게 찾아내어 추천 정확도가 자연스럽게 향상됩니다.
